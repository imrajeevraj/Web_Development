1. Inline CSS

Inline CSS is applied directly to HTML elements using the style attribute.

This is Inline CSS!

Every style is defined directly in the HTML tag.

<div style="background-color: #4facfe; padding: 20px;">
  <h3 style="margin: 0; color: white;">Inline CSS</h3>
</div>

Pros: Quick for single elements, highest specificity

Cons: Hard to maintain, not reusable, mixes content with presentation

2. Internal CSS

Internal CSS is defined within the <style> tag in the HTML document's <head> section.

This uses Internal CSS!

These styles are defined in the <style> tag at the top of this HTML file.

They can be reused throughout this single page.

<style>
  .internal-example {
    background: linear-gradient(135deg, #667eea 0%, #764ba2 100%);
    color: white;
    padding: 16px;
  }
</style>

Pros: Reusable within the page, keeps CSS organized, good for single-page sites

Cons: Not reusable across multiple pages, increases HTML file size

3. External CSS

External CSS is defined in a separate .css file and linked using the <link> tag.

This uses External CSS!

Normally loaded from a separate file: external.css

<!-- In HTML head -->
<link rel="stylesheet" href="external.css">

/* In external.css file */
.external-box {
  background: linear-gradient(to right, #f093fb 0%, #f5576c 100%);
  padding: 24px;
}

Pros: Reusable across multiple pages, keeps HTML clean, cached by browser, easy to maintain

Cons: Requires additional HTTP request, may load unused styles
